## Task Manager — Java Desktop Application

###  1. Обзор
Task Manager — это настольное Java-приложение для управления пользовательскими задачами, ориентированное на простоту использования, автономность и строгое соблюдение принципов объектно-ориентированного программирования.  
Приложение создано как учебный проект, имеющий полнофункциональную архитектуру, аналогичную промышленным системам:  
раздельные слои (Model–Service–Storage–UI–Utils)  
гибкое расширение  
логирование  
обработка ошибок  
поддержка нескольких пользователей  
### 2. Ключевые функции  
#### **УПРАВЛЕНИЕ ЗАДАЧАМИ**
- Создание задач с названием, описанием, приоритетом и дедлайном
- Удаление ненужных задач
- Отметка задач как выполненных
- Автоматическое определение просроченных задач
- Сортировка по приоритету и дате

#### **СИСТЕМА ПОЛЬЗОВАТЕЛЕЙ**
- Регистрация новых пользователей
- Авторизация существующих пользователей
- Выделенные файлы-хранилища для каждого пользователя

####� **ГРАФИЧЕСКИЙ ИНТЕРФЕЙС**
- Современная тема FlatLaf (темный стиль)
- Вкладки: Все задачи / Выполненные / Просроченные
- Кнопки управления и диалоговые окна
- Интуитивное добавление и редактирование задач

#### **ХРАНЕНИЕ ДАННЫХ**
- Локальные текстовые файлы (без интернета)
- Раздельные хранилища: задачи и пользователи
- Автономная работа приложения

#### **ЛОГИРОВАНИЕ (Log4J)**
- Логирование ошибок и успешных действий
- Отслеживание входов пользователей
- Фиксация создания и изменения задач

#### **ОБРАБОТКА ОШИБОК**
- Защита от некорректного ввода данных
- Обработка ошибок файловой системы
- Безопасное завершение программы
### 3. Архитектура системы
Приложение построено по модульной 5-слойной архитектуре:  
src/  
 ├── model/     - бизнес-сущности  
 ├── service/   - логика управления  
 ├── storage/   - файловое хранилище  
 ├── ui/        - графический интерфейс  
 └── utils/     - вспомогательные классы  
Это обеспечивает:  
слабую связность  
лёгкую заменяемость компонентов  
возможность расширять приложение без переписывания ядра

### НАСЛЕДОВАНИЕ 
Наследование применено там, где оно приносит архитектурную пользу:

```  
// Кастомная модель таблицы, расширяющая стандартную
public class TaskTableModel extends AbstractTableModel {
    private List<Task> tasks;
    private String[] columnNames = {"Название", "Приоритет", "Дедлайн", "Статус"};
    
    @Override
    public int getRowCount() { return tasks.size(); }
    
    @Override
    public int getColumnCount() { return columnNames.length; }
    
    @Override
    public Object getValueAt(int row, int col) {
        Task task = tasks.get(row);
        switch (col) {
            case 0: return task.getTitle();
            case 1: return task.getPriority();
            case 2: return task.getDeadline();
            case 3: return task.isCompleted() ? "yes" : "no";
            default: return "";
        }
    }
    
    @Override
    public String getColumnName(int col) {
        return columnNames[col];
    }
}
```
