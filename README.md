## Task Manager — Java Desktop Application

###  1. Обзор
Task Manager — это настольное Java-приложение для управления пользовательскими задачами, ориентированное на простоту использования, автономность и строгое соблюдение принципов объектно-ориентированного программирования.  
Приложение создано как учебный проект, но имеет полнофункциональную архитектуру, аналогичную промышленным системам:  
раздельные слои (Model–Service–Storage–UI–Utils)  
гибкое расширение  
логирование  
обработка ошибок  
поддержка нескольких пользователей  
### 2. Ключевые функции  
✔ Управление задачами

автоматическое определение просроченных задач  
сортировка задач по приоритету и дате

✔ Хранение и устойчивость  
локальные текстовые файлы

### 3. Архитектура системы
Приложение построено по модульной 5-слойной архитектуре:  
src/  
 ├── model/     - бизнес-сущности  
 ├── service/   - логика управления  
 ├── storage/   - файловое хранилище  
 ├── ui/        - графический интерфейс  
 └── utils/     - вспомогательные классы  
Это обеспечивает:  
слабую связность  
лёгкую заменяемость компонентов  
возможность расширять приложение без переписывания ядра

### 5. Подробное описание модулей
#### `Task.java` – Сущность задачи
- **Название** (title) – текстовое название задачи
- **Описание** (description) – подробное описание задачи
- **Приоритет** (priority) – числовой приоритет от 1 до 5
- **Дедлайн** (deadline) – дата окончания срока выполнения
- **Состояние** (completed) – флаг выполнения задачи
- **Методы проверки просрочки** – автоматическое определение просроченных задач

#### `User.java` – Сущность пользователя
- **Логин** (username) – уникальное имя пользователя
- **Пароль** (password) – зашифрованный пароль пользователя
- **Методы проверки валидности** – валидация учетных данных

###  **PAKAGE: SERVICE** (Бизнес-логика)

#### `TaskManager.java` – Реализация ключевой бизнес-логики
- **Добавление / удаление задач** – управление жизненным циклом задач
- **Сортировка** – упорядочивание задач по различным критериям
- **Выборка выполненных / просроченных** – фильтрация по статусу задач
- **Обновление статуса** – изменение состояния выполнения задач
- **Использует Java Streams & Comparator** – для эффективной обработки коллекций

###  **PAKAGE: STORAGE** (Хранение данных)

#### `FileStorage.java` – Работа с файловым хранилищем
- **Чтение задач из файла** – загрузка данных из текстового файла
- **Запись задач** – сохранение данных в текстовый файл
- **Сериализация / десериализация** – преобразование объектов в строки и обратно

#### `UserStorage.java` – Управление пользовательскими данными
- **Проверка существования пользователя** – поиск пользователя в базе
- **Валидация авторизации** – подтверждение правильности учетных данных

###  **PAKAGE: UI** (Пользовательский интерфейс)

#### `SwingApp.java` – Основной класс графического интерфейса
- **Главное окно** – центральное окно приложения
- **Вкладки** – разделы для разных типов задач
- **Обработчики кнопок** – управление действиями пользователя
- **Вызов диалоговых окон** – окна для ввода данных
- **Отображение таблиц** – визуализация списка задач

### **PAKAGE: UTILS** (Вспомогательные утилиты)

#### `LoggerUtil.java` – Настройка системы логирования Log4J
- **Паттерн логов** – формат записи лог-сообщений
- **Уровни логирования** – INFO, DEBUG, WARN, ERROR
- **Директория хранения** – путь к файлам логов (`logs/app.log`)
### 6. Формат хранения данных
users.txt:  
username:passwordHas  
tasks_username.txt:  
title|description|priority|deadline|completed  
title2|desc2|3|2025-02-11|false  
Простое хранение, позволяющее без труда расширить до JSON или SQLite.  

### 7. Принципы ООП в деталях  
✔ Инкапсуляция  
Все поля закрыты, доступ только через геттеры/сеттеры.  
✔ Полиморфизм  
Используется для сортировок:  
Comparator<Task> byPriority  
Comparator<Task> byDeadline  
✔ Абстракция  
Чёткое выделение слоёв.  
✔ Наследование  
Модели подготовлены к расширению для PremiumUser или Admin.  
### 8. Обработка исключений
Используются:  
try/catch  
обработка ошибок ввода  
проверка формата дат  
проверка существования файлов  
Фатальные ошибки логируются, но не крашат приложение.  
### 9.  Система логирования  
Логи лежат в:  
logs/app.log  
Записываются:  
вход пользователя ;регистрация ;создание задачи ;удаление ;ошибки ввода ;ошибки чтения файлов.

Компиляция проекта:

### На Windows
